# Pilot DPA Checklist

Use this before onboarding each pilot customer.

## Customer and scope

- Customer legal entity and contact confirmed.
- Business Central environment type confirmed: sandbox or production.
- Pilot scope confirmed: Quick Scan, Deep Scan, Analytics Dashboard, manual support.
- Automatic correction actions excluded unless separately agreed.

## Data processing

- Categories of processed data documented.
- Backend stores aggregated scan findings and status data, not full record exports.
- Potential personal data in duplicate findings explained.
- Hosting region and subprocessors documented.
- Stripe processing clarified if billing is enabled.

## Security

- Tenant registration invite code issued per pilot.
- API token returned once and stored in Business Central isolated storage.
- Admin access limited to named operators.
- Support access and incident contact documented.

## Retention and deletion

- Pilot retention period agreed.
- Manual export process agreed.
- Manual deletion process agreed.
- Billing/audit retention exceptions explained.

## Legal/commercial

- Pilot agreement signed.
- DPA/AVV signed or explicitly covered by pilot agreement.
- Liability and no-guaranteed-savings language included.
- Pilot feedback and escalation cadence agreed.
